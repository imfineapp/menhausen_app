# Deno Configuration for Edge Functions

This directory contains Supabase Edge Functions that run on Deno runtime, not Node.js.

## TypeScript Checking

These functions are **excluded** from the main TypeScript configuration (`tsconfig.json`) because they use:
- Deno-specific imports (e.g., `https://deno.land/std@0.168.0/http/server.ts`)
- Deno global objects (e.g., `Deno.crypto`)
- Different module resolution system

## Running Type Checks

Edge Functions are type-checked separately by:
1. **Supabase CLI**: `supabase functions deploy` performs type checking
2. **Deno CLI**: Can use `deno check` if Deno is installed locally
3. **IDE Support**: Configure your IDE to use Deno language server for `supabase/functions/` directory

## Local Development

When developing Edge Functions:
- Use `supabase functions serve` to test locally
- The Supabase CLI will handle type checking during deployment
- Deno-specific TypeScript errors are expected when viewing in Node.js-focused IDEs

## Note

The main `npm run type-check` command intentionally excludes this directory to avoid Deno-related type errors in the Node.js/React application context.
